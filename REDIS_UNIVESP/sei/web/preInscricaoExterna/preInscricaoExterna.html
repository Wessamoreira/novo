Instruções:
* Este é um html de exemplo a ser utilizado pelo cliente.
* Deve ser garantido que as informações da propriedade name dos componentes sejam mantidos.
* A posição dos campos, fontes, cores e etc podem ser alterados conforme a necessidade do cliente.
* Existem 3 campos ocultos chamados unidade, curso e turno que devem ser informado no mesmo o código deste no SEI, o único que é opcional é o campo turno.
* O campo CPF é opcional desde que seja marcado na configuração padrão do sistema para não validar o CPF na pré-inscrição.
* Os campos Nome, Email, um do Telefones são obrigatorios.
* Os campos Cidade, Naturalidade devem ser alterado as propriedades options para buscar a informação conforme a base de dados do cliente:
* Caso na configuração padrão do sistema esteja marcado que todos os campos da pré-inscrição é obrigatório então as demais informações serão validadas.
* Na tag form do html deve ser informado na tag action a url do SEI seguido de /preinscricaoservice. Ex: www.sei.edu.br/preinscricaoservice
* O funcionamento deste depende da criação de uma campanha de pré-inscrição no crm, bem como a definição do funcionário responsável para cada curso e demais configurações a serem tratadas com o suporte
